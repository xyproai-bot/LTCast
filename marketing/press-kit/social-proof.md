# Social proof — to be populated after launch

This file is deliberately sparse at time of launch. Fill it in as real quotes, reviews, and case studies arrive.

Rules:

- **No invented quotes.** Ever. Not even "representative" ones.
- Every quote needs a source URL or written permission on file.
- If a user grants permission verbally, follow up in writing (email) before publishing. Store the written permission in `press-kit/permissions/` (not committed).
- Quotes from private Slacks / Discords are off-limits unless explicitly cleared.

## Case studies — longer form

Reserve this section for 300–500 word case studies. A case study includes:

- Show / production / venue name (with permission)
- Problem statement — what was wrong with the previous workflow
- How LTCast was used — which protocols, which hardware, which team members
- Measurable outcome — setup time saved, cues not dropped, crew size reduced, etc.
- Permission on file — date + email excerpt quoted

Case study candidates to pursue:
- A touring pro running a Windows show PC across a 20-date tour
- A community theatre replacing a hardware LTC box
- A corporate AV rental house standardising a Windows show-PC image across jobs
- A worship tech team syncing lights to a click track
