# Launch posts

Drafts for the initial launch wave. Each file is platform-specific — do **not** copy-paste between them. Different audiences react badly to the same copy.

## Posting order and spacing

Sequence matters. Each platform has its own rhythm; posting all of them in one day looks like spam and collapses traction on every individual post.

| Day | Channel | File |
|-----|---------|------|
| T-0 | Product Hunt (00:01 PT) | [product-hunt.md](product-hunt.md) |
| T+0 | Hacker News Show HN (around 08:00 UTC same day) | [hackernews.md](hackernews.md) |
| T+2 | r/lightingdesign | [reddit-lightingdesign.md](reddit-lightingdesign.md) |
| T+4 | ControlBooth (Commercial Announcements) | [controlbooth.md](controlbooth.md) |
| T+5 | ETC Community (Eos Family subforum) | [etc-community.md](etc-community.md) |
| T+7 | r/techtheatre | [reddit-techtheatre.md](reddit-techtheatre.md) |
| T+10 | r/livesound (only if we have standing — otherwise skip) | [reddit-livesound.md](reddit-livesound.md) |

Rationale for the spacing:

- Product Hunt and HN share an audience — day-of-launch is fine, but the HN post should lead with a technical frame (not the PH comments).
- The three Reddit posts target overlapping-but-distinct subs. Cross-posting the same content is a bannable pattern on Reddit. Each post is written specifically for its sub.
- ControlBooth and ETC Community are forums; their posts live for years and should read like reference documents, not announcements.
- r/livesound is the hardest crowd. Only post there if the account has real standing in the sub — otherwise the post will be removed and the account burned.

## Ground rules for every post

- **No cross-pollinating copy.** Each platform's post should read as if it was written for that platform only.
- **No AI-voice.** Avoid "unlock", "revolutionary", "game-changer", "seamless", "delightful", "powerful". Be concrete.
- **No price-led leads.** Lead with the problem or a specific technical detail. Price comes in the FAQ.
- **Reply within the first 30 minutes** on every platform. Conversion rate is dominated by how fast the maker responds to the first five comments.
- **Never DM-beg for upvotes / comments.** Product Hunt, HN, and Reddit all have sophisticated detection for this and the penalties are permanent.
- **Never respond defensively.** If someone trashes the product, thank them for the specific critique and move on. If someone asks a question you can't answer, say "don't know yet — testing this week, will report back" and follow through.

## What's deliberately missing

- **Twitter / X.** Low ROI for a niche pro tool. A single launch thread is fine; full campaigns aren't worth the effort here. If we do one, post it as a reply to the HN thread link, not as a standalone.
- **LinkedIn.** Only if the maker already has an AV / theatre-industry network there. Generic LinkedIn audience won't convert.
- **YouTube.** The demo video exists (see `../demo-video-script.md`) — post it once on the landing page, once pinned on the YouTube channel, and embed it in the HN and ControlBooth posts. Don't do a separate "YouTube launch".
- **Facebook groups.** There are several large lighting / touring AV groups on FB. They're a follow-up channel after launch week, not part of it. Each has its own posting rules — enumerate them separately before going in.

## Measurement

After launch week, log actual outcomes in `../press-kit/social-proof.md`:

- PH rank + comment count + most useful maker reply
- HN peak position + total points + best comment thread to quote
- Reddit upvote ratios and best-received comments
- Forum reply counts and any threads that hit the "answered" / "pinned" state

Quotes from real threads become the best copy on the landing page and in the next round of press outreach. Hunt them actively.
